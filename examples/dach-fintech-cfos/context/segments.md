# Segments — tiers & scoring rubric

## Segment A (primary — pursue aggressively)
200–2,000 employees, fintech in DACH, Series B+ funding, has a dedicated finance function.

## Segment B (adjacent — pursue selectively)
50–200 employee regulated payments / lending / treasury startups in DACH.

## Segment C (exclude / deprioritize)
<50 employees, no finance function, crypto-native with no fiat controls.

## Scoring rubric (0–10)
- Persona match: 0–4   (exact CFO/Compliance title+seniority = 4; adjacent = 2; weak = 0)
- Company fit:   0–3   (Segment A = 3; B = 1–2; C = 0)
- Data completeness: 0–2   (has linkedin_url + company_domain = 2)
- Novelty:       0–1   (not already in CRM = 1)

## Thresholds
- QUALIFY  >= 7
- MAYBE    4–6
- SKIP     <= 3
